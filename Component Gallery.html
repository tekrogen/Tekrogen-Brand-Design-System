Every component, with the code beside it.

The kit's components — the five primitives plus the C1 structural composites (header, footer, section-head, label) — in the token-only components/ registry per ADR-0011. Each block shows a live, theme-following specimen and a tabbed ⧉ copy control: HTML (the markup contract), CSS (the .tk-* token block), and Ghost (the same markup as a Handlebars partial for the future theme).

Method & scope

Source: the five already-shipped primitives in index.html (Button, Card, Input, Badge — the mono pill — and Avatar), extracted render-for-render into components/<name>/ as .tk-* classes that consume var(--tk-*) only. Off-scale padding was snapped to the --tk-space-* scale (a small, intended pixel-diff logged per ADR-0003); type, radius, and hairlines map to existing tokens. The roadmap's remaining components slot into the six families in Component Gap Analysis.html; Header / Footer / Navigation follow next.

One-cyan caveat: a catalog deliberately shows several cyan-primary variants at once. Per ADR-0011, a component offers a variant; the page it lands on decides which single instance gets the brand's one cyan accent — so the multiplicity here is documentation, not a brand violation.

Version label: the jun 2026 stamp is static and intentionally not registered in scripts/version-stamp.mjs (FILES), matching the two study pages — so stamp-check does not guard this surface.

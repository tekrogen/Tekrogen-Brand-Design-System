Type Size Remediation

All font sizes below 12px raised to meet readable-text minimums across the token layer and all UI kit components.

Completed · Jun 6, 2026
Standards Applied

Body text

16–20px
Paragraph text minimum. Tekrogen uses 16px — ✓ already correct.

Secondary / small text

14px
~2 sizes below body. Tekrogen --tk-fs-body-sm = 14px — ✓ correct.

Form inputs

16px
Prevents iOS auto-zoom. Subscribe email input raised from 14 → 16px.

On eyebrow & meta labels (12px floor)

The --tk-fs-eyebrow and --tk-fs-meta tokens serve a structural role — uppercase, widely-tracked mono labels used as section kickers, breadcrumbs, and metadata chips. These are not reading text; they are orientation glyphs.

WCAG 2.1 sets no hard pixel-size floor, but general guidelines (pimpmytype.com, learnui.design) cite 14px for readable prose. For all-caps mono with 0.12–0.16em letter-spacing, 12px delivers equivalent legibility to ~14px mixed-case text. The previous 10–11px values fell below any defensible threshold. 12px is the practical minimum for this role.

If future user research shows these labels are difficult to read, the next step is raising them to 13px — not converting them to mixed case, which would break the visual hierarchy.

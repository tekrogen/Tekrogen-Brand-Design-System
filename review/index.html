00 · SUMMARY

Strong foundations. Thin governance. Both fixable in the same pass.

Scope · this repo
External flags · BNR, WF-001

The analysis read the system fairly. The pillar semantics are real, the tokens are doing real work, and the writing voice has the procedural honesty the brand stakes itself on. The places it called thin — single-source palette, sans/serif drift, the absence of a paid-flow mock — were thin.

Two claims were overstated. The serif drift was real in the documentation but the shipping pixels were already sans; no editor was ever going to read a serif page. And the BNR / WF-001 inconsistencies are real, but they live in workflow and roadmap docs outside this repo — they're flagged here, not fixed here.

This pass single-sources the palette through tokens/palette.js, scrubs the stale serif language, publishes six ADRs as the canonical record of load-bearing decisions, and adds trust-state-matrix.html so the paid flow has a place to live before the first real payment ever happens.

01

Typography source-of-truth split.

Real · softened
Analyst's claim

"colors_and_type.css declares sans-only; preview/_card.css pulls IBM Plex Serif and Source Serif 4; tekrogen-org/README says serif body. Editors and engineers will ship conflicting reading experiences."

What the code actually says

Half right. Shipping pixels were already 100% Poppins — no JSX file actually applied a serif font-family. But the dead serif imports were real, and the stale comments would have confused a new contributor.

Action taken

Dropped IBM+Plex+Serif and Source+Serif+4 from preview/_card.css. Updated 4 stale "serif" references (README, two JSX header comments, the Plex Sans specimen). Codified the stance as adr/0001. In v0.3.0, Plex Sans was retired entirely — the woff2, the @font-face, and the specimen card are gone.

ADR-0001 · preview/_card.css · ui_kits/tekrogen-org/README.md · Article.jsx · FieldNoteCard.jsx · colors_and_type.css · fonts/README.md
02

Duplicate color authority.

Fixed
Analyst's claim

"Pillar hex values live in marks.jsx (TK_PALETTE) and colors_and_type.css — must stay manually synchronized. Visual regression risk when one file updates."

What the code actually says

Worse than the analyst said. Pillar hexes were duplicated in seven places: the CSS, marks.jsx, Dragonfly.jsx, Article.jsx (a hardcoded gradient), master-lockups/index.html, asset-pack/index.html, and the SVG assets.

Action taken

New tokens/palette.js is the single JS source. tokens/sync.mjs mirrors it into the CSS file between markers; --check mode exits non-zero on drift (CI hook). Consumers now read from window.TK_TOKENS and throw if it's missing.

ADR-0002 · tokens/palette.js · tokens/sync.mjs · colors_and_type.css · marks.jsx · Dragonfly.jsx · Article.jsx · master-lockups · asset-pack

04

BNR ↔ live stack drift.

External · flagged
Analyst's claim

"BNR .org row references Next.js MDX as canonical; WF-001 asserts Ghost Pro live. Teams misplace canonical URL and rendering — SEO and design parity diverge silently."

What the code actually says

Accurate. The design system commits to Ghost Pro as the rendering surface for .org — but the BNR roadmap still describes MDX as canonical. Two contradictions of the same fact.

Action taken

Pinned the DS assumption in adr/0004: Ghost Pro is canonical here. The BNR amendment is recommended but not in this repo's gift to write.

06

Trust-stage UX not modelled in-repo.

Fixed
Analyst's claim

"Subscribe form fakes success. Member / paid / entitled flows not mirrored. First real payment will collide with placeholder optimism — credibility shock."

What the code actually says

Correct and the most strategically important call in the analysis. The DS modelled the happy path only; the four real reader states had no visual contract.

03 · ADRs

Six load-bearing decisions, written down once.

All accepted
2026-05-16
ADR · 0001

Typography — sans-only.

Poppins is the only face. Editorial weight comes from size, tracking, and italic — not from a serif counterpoint.

Accepted●
ADR · 0002

Palette — single source.

tokens/palette.js is canonical. sync.mjs mirrors it into CSS. Drift detectable in CI.

Accepted●
ADR · 0003

Token versioning — semver, even pre-1.0.

Pre-1.0 minor = any visual change. Discipline starts now so consumers don't pull surprises.

Accepted●
ADR · 0004

Canonical authoring — Ghost Pro.

DS assumes Ghost Pro. BNR drift flagged for external amendment.

Accepted●
ADR · 0005

Icon system — Lucide on product, glyphs on brand.

Outline 1.5 stroke, currentColor, four-step size scale. Brand surfaces stay icon-free.

Accepted●
ADR · 0006

Trust-state CTAs — paired-content primitive.

Anonymous / member / paid / entitled. One cyan accent, four contracts.

Accepted●

06 · OPEN QUESTIONS · RESOLVED

v0.3.0 stakeholder pass
OQ · 01 · RESOLVED

The .studio verb-space.

Decided · "Proof of concept and demos." Propagated across Footer, lockup sheet, App-shell, README. The "learning" framing is retired; "demos / proof" is now the canonical positioning.

OQ · 02 · RESOLVED

IBM Plex Sans fate.

Decided · Retired. The self-hosted woff2, the @font-face, the Google Fonts URL fragment, and the specimen card are all gone. Tekrogen is sans-only via Poppins. A future second sans needs a new ADR.

OQ · 03 · ON HOLD

Package boundary — @tekrogen/tokens.

Decided · On hold. When .com or .net consumes tokens this becomes load-bearing; until then the two-file / one-script contract in tokens/ is sufficient. ADR-0003 holds the semver discipline; a future ADR will spin out the package.

A brand that earns attention through procedure should be evaluated procedurally. The Cursor analysis read fairly; this pass closed the cheap fixes, wrote down the structural ones, and answered the three deferred questions before they could drift.
